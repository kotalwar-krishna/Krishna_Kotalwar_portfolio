Object In Javascript

Objects are used to store keyed collections of various data and more complex entities. In JavaScript, objects penetrate almost every aspect of the language.An object can be created with figure brackets “{}” with n optional list of properties. A property is a pair of Key and value (key: value)

One object creating by syntax

let user = new Object(); // "object constructor" syntax
            let user = {};  // "object literal" syntax

Object Literal

let user = {     
            name: "John",  
            age: 30        
          };

In this object “name” is key, the key is a string and “john” is value, a value can be anything. In the second property value is Number.

Property values are accessible using the dot notation:

console.log( user.name ); // John
            console.log( user.age ); // 30

We can do add & remove properties:

user.isAdmin = true;
            delete user.age

If we want to add multiword-property names, they must be written in quoted:

let user = {     
            name: "John",  
            age: 30 ,
          "Pokey the Penguin" : "Steve Havelka" 
          "sachin god of cricke" : true     
          };

For multiword properties, the ‘dot’ access doesn’t work. Then we used Square Brackets.

Square Brackets

user.likes animals= true // syntax error

The above equation gives a syntax error because it thinks that we address user.likes, unexpected animals. The dot requires the key to be a valid variable identifier. That implies: contains no spaces, doesn’t start with a digit, and doesn’t include special characters.

There’s an alternative “square bracket notation” that works with any string:

user['likes animals']= true

Square brackets are much more powerful than the dot notation. They allow any property names and variables. But they are also more cumbersome to write. So most of the time, when property names are known and simple, the dot is used. And if we need something more complex, then we switch to square brackets.

The key and value are the same then we can write a shorthand property.

let user = {
            name,  // same as name:name
            age: 30
          };

Property names limitation

Property names (keys) must be either strings or symbols. Other types are automatically converted to strings.

Reserved words are allowed as property names.

The variable cannot have a name equal to one of the language-reserved words like “for”, “let”, “return” etc. But for an object property, there’s no such restriction.

let obj = {
            for: 1,
            let: 2,
            return: 3
          };
          
          console.log( obj.for + obj.let + obj.return );
